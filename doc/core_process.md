Tauri、`wgpu`、FFmpeg、カスタムプロトコルを組み合わせた「最小パイプライン（フェーズ1：単一フレーム描画）」を構築するための実装仕様書です。

この仕様書のゴールは、**「フロントからシーク位置を受け取り、Rust側でデコード・合成した1フレームを、カスタムプロトコル経由でフロントの `<img>` タグに表示する」** という一連のライフサイクルを成立させることです。

---

## 1. 全体アーキテクチャ・データフロー

```
[フロントエンド (UI)]
   │ 
   │ ① シーク位置 (例: timestamp: 3000ms) を送信
   ▼ (Tauri IPC: Invoke)
[Tauri コア (Rust)]
   │
   │ ② タイムスタンプをデコーダへ
   ▼
[FFmpeg デコーダ] ──(③ Rawピクセルバッファ: RGBA)──> [wgpu レンダラー]
                                                               │
                                                               │ ④ レンダーグラフ合成 &
                                                               │    オフスクリーンレンダリング
                                                               ▼
[カスタムプロトコル] <──(⑥ JPEG/PNG バイト列)── [イメージエンコーダ]
   │
   │ ⑤ HTTPレスポンス
   ▼ (custom://)
[フロントエンド (<img>)]

```

---

## 2. 構成要素と技術スタック

* **フロントエンド:** HTML5 / TypeScript (Vite + Vanilla JS または React/Vue)
* **バックエンド (Tauri):** Tauri v2 (Rust)
* **ビデオデコーダ:** `ffmpeg-next` クレート (または `rsmpeg`)
* **グラフィックス/合成:** `wgpu` クレート
* **画像エンコーダ:** `image` クレート（高速なJPEGまたはPNGエンコード用）

---

## 3. 各モジュールの詳細仕様

### 3.1. フロントエンドモジュール

* **役割:** タイムラインUIを提供し、シーク時にRustへイベントを通知、返ってきた画像を画面に描画する。
* **仕様:**
1. `<input type="range">`（シークバー）と `<img id="preview-canvas">` を配置。
2. シークバーが操作されたら（`input` イベント）、Tauriの IPC コマンド `invoke("seek_frame", { timeMs: value })` を呼び出す。
3. **[重要]** IPCのレスポンスとしては画像バイナリそのものではなく、**カスタムプロトコルのURL（タイムスタンプをクエリパラメータに含む）** を受け取る。
* 例: `video-preview://localhost/frame?t=3000`


4. 受け取ったURLを `<img>` タグの `src` に代入して画像を更新する。



### 3.2. Tauri IPC コマンド (`commands.rs`)

* **役割:** フロントエンドからのシーク要求を受け付け、プレビュー用のURLを即座に返す。
* **仕様:**
```rust
#[tauri::command]
fn seek_frame(time_ms: u64, state: tauri::State<'_, AppState>) -> Result<String, String> {
    // 1. Rust側の状態（現在のシーク位置）を更新
    let mut current_time = state.current_time.lock().unwrap();
    *current_time = time_ms;

    // 2. フロントエンドにリクエストすべきカスタムプロトコルのURLを返す
    // キャッシュを避けるためタイムスタンプをクエリに付与
    Ok(format!("video-preview://localhost/frame?t={}", time_ms))
}

```



### 3.3. FFmpeg デコードモジュール (`decoder.rs`)

* **役割:** 指定されたタイムスタンプの動画フレームをRawピクセルとして取り出す。
* **仕様:**
1. 動画ファイルを読み込み、ビデオストリームを特定する。
2. `seek_frame` 指示が来たら、指定された `time_ms` に一番近いキーフレームにシークする。
3. 目的のフレームに到達するまでデコードを順方向に進める。
4. 取り出したフレーム（通常はYUVフォーマット）を、`wgpu` が扱いやすい `RGBA`（または `BGRA`）フォーマットに変換し、`Vec<u8>` のバッファとして出力する。



### 3.4. wgpu レンダリングモジュール (`renderer.rs`)

* **役割:** デコードされたピクセルをGPUに転送し、合成（今回は最小構成のため色反転などのモックエフェクト）を行い、オフスクリーンバッファに書き出す。
* **仕様:**
1. **初期化:** `wgpu::Instance` から `wgpu::Adapter`、`wgpu::Device`、`wgpu::Queue` を作成。ウィンドウを持たないため、Surfaceの作成は不要。
2. **テクスチャ作成:** デコードされた動画サイズ（例: 1920x1080）の `wgpu::Texture`（入力用）と、同じサイズのテクスチャ（出力・レンダリングターゲット用）を作成。
3. **データのアップロード:** `queue.write_texture` を使い、FFmpegから得たRawピクセルを入力用テクスチャに転送。
4. **レンダーグラフ（シェーダー）実行:** 入力テクスチャをサンプリングし、出力テクスチャへ描画するパイプラインを実行。（※最小構成では、テクスチャをそのままコピーする、または色を反転させるだけのフラグメントシェーダーで可）。
5. **データの読み出し:** 出力テクスチャから `wgpu::Buffer` へデータをコピー（`copy_texture_to_buffer`）し、GPUメモリからメインメモリ（CPU）へマップして `Vec<u8>`（RGBAバッファ）として回収する。



### 3.5. カスタムプロトコルハンドラ (`protocol.rs`)

* **役割:** `<img>` タグからの画像要求に対し、その場でwgpuを回して生成した画像バイナリをHTTPレスポンスとして返す。
* **仕様:**
1. `tauri::Builder` の `.register_asymmetric_uri_scheme_protocol`（または `register_uri_scheme_protocol`）で `video-preview` プロトコルを登録。
2. リクエスト（URL）から `t=XXXX` のタイムスタンプを解析。
3. **同期・実行パイプラインの呼び出し:**
* `decoder.rs` を呼び出し、該当フレームをデコード。
* `renderer.rs` を呼び出し、`wgpu` でオフスクリーンレンダリング。


4. 得られたRGBAの生バッファを、`image` クレート等を使って高速に `JPEG` フォーマットにエンコードする。
5. レスポンスの `Mime-Type` を `image/jpeg` に設定し、バイナリデータを返す。

## 4. 開発・検証のステップ

まずは複雑な動画シークを後回しにし、以下の順でパイプラインを「下流から」デバッグしていくのが定石です。

1. **ステップ1 (カスタムプロトコル ⇄ フロント単体テスト)**
* Rust側で静的な1枚のJPEG画像を読み込んでおき、カスタムプロトコル経由でフロントの `<img>` に表示できるか確認する。


2. **ステップ2 (wgpu ⇄ カスタムプロトコル連携)**
* Rust側で `wgpu` を起動し、緑単色やグラデーションなどの「ダミーテクスチャ」をオフスクリーンレンダリングし、それをJPEG化してカスタムプロトコルでフロントに表示できるか確認する。


3. **ステップ3 (FFmpeg ⇄ wgpu 連携の完成)**
* 動画の「最初の1フレーム」だけを固定でデコードし、それを `wgpu` に投げてフロントに表示できるか確認する。


4. **ステップ4 (シークの実装)**
* フロントのシークバーの動きに合わせて、指定タイムスタンプの動画フレームが動的に切り替わるように処理を繋ぎ込む。

5. **ステップ5 (実際の動画ファイルでの検証)**
* 実際の動画ファイルを読み込み、シークバーを動かすと動画のプレビューが正しく更新されるか確認する。読み込みはドラッグ＆ドロップで行えるようにするのが望ましい。
